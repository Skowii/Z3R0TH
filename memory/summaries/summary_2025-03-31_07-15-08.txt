El patrón de módulos se utiliza para implementar el concepto de módulos de software en un lenguaje de programación con soporte directo incompleto. Los componentes de software, como la base de datos, la interfaz de usuario y la API, se desacoplan utilizando un gestor de módulos que actúa como un mercado central para registro de servicios y solicitudes entre ellos. Esto garantiza que los cambios en uno de ellos no afecten al sistema. El patrón Módulo encapsula funciones y datos relacionados en una unidad, permitiendo componentes organizados y manejables. Su implementación varía dependiendo del lenguaje de programación anfitrión. En Python y Ada, cada archivo .py y paquete respectivamente se considera un módulo.
