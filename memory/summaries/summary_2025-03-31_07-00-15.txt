El patrón del controlador frontal se utiliza para introducir un gestor común para todas las solicitudes de un sitio web, permitiendo encapsular funcionalidad común como la seguridad, internacionalización, enrutamiento y registro en un solo lugar. Este patrón es aplicable cuando se desea encapsular la funcionalidad común de gestión de peticiones en un único lugar o implementar la gestión dinámica de peticiones sin modificar el código. Además, hace la configuración del servidor web portable solo necesitando registrar el manejador específico para el servidor web. Ejemplos del mundo real incluyen Apache Struts.
