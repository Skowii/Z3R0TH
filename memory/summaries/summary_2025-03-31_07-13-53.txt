El patrón de diseño Client Session se utiliza para mantener el estado y los datos de un usuario a través de múltiples peticiones en una aplicación web, ofreciendo una experiencia de usuario continua y personalizada. Los servidores no almacenan información del cliente, ya que estos proporcionan toda la información necesaria para realizar el proceso en cada solicitud. Se utilizan objetos Session para almacenar información relevante y pasarla con cada Request al servidor.
