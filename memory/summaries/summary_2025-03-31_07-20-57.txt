Flyweight es un patrón de diseño utilizado para compartir objetos similares y reducir el consumo de memoria o gastos computacionales. Se aplica cuando se necesita representar múltiples elementos con una única instancia de objeto. Este enfoque permite minimizar el uso de recursos al compartir datos entre objetos similares. Un ejemplo práctico sería un inventario de pociones mágicas en una tienda, donde se crean pociones en lugar de tener una instancia para cada una, lo que reduce la huella de memoria. La clase Flyweight `PotionFactory` es responsable de crear y gestionar las pociones, utilizando una estructura EnumMap para compartir datos entre objetos similares.
