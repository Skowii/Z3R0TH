Context Object es un patrón creacional que se utiliza para desacoplar datos específicos de protocolos y almacenarlos en objetos independientes del protocolo subyacente. Permite compartir información eficientemente a través de varias capas de la aplicación. Se define un objeto de contexto (Context Object), como ServiceContext, que contiene datos relevantes para su uso en diferentes partes del software. La interfaz ServiceContextFactory se utiliza para crear objetos de contexto en diferentes capas de la aplicación.
