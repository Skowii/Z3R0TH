Event Sourcing는 데이터의 현재 상태만 도메인에 저장하고, 추가 전용 저장소를 사용해 해당 데이터에 수행된 전체 작업을 기록합니다. 저장소는 도메인 객체를 구체화하며, 데이터 모델과 비지니스 도메인의 동기화가 필요 없어 성능, 확장성, 응답성이 향상됩니다. 트랜잭션 데이터에 대한 일관성을 제공하고, 전체 감사용 기록과 히스토리를 유지 관리할 수 있습니다.

이벤트 소싱 패턴은 관계형 데이터 구조가 복잡한 어플리케이션 상태를 유지하기 위해 높은 성능이 필요한 경우, 특정 시점의 상태를 복원할 수 있는 로그가 필요한 경우, 과거에 발생한 이벤트를 다시 적용하여 프로덕션 문제 디버깅에 사용될 수 있습니다. 실제 적용 사례와 리소스는 지원서를 참조합니다.
