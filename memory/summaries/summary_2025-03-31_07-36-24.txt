El patrón de fábrica, también conocido como "Simple Factory" o "Static Factory Method", proporciona un método estático encapsulado en una clase llamada fábrica para ocultar la lógica de implementación y centrar el código del cliente en el uso en lugar de inicializar nuevos objetos. Este patrón permite crear objetos diferentes sin modificar el código fuente existente y proporciona un solo punto para la creación de objetos y gestión, lo que resulta en código poco acoplado con mejores ventajas como testabilidad y mantenibilidad.
