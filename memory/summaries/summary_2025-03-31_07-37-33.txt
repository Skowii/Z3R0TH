El patrón Special Case se utiliza para definir casos especiales y encapsularlos en subclases que proporcionan diferentes comportamientos especiales. Se enfoca en devolver objetos reales no nulos con comportamientos especiales. Este patrón puede ser útil en el mundo real, como en un sistema de comercio electrónico para devolver un modelo de vista de recibo que contenga datos reales de la compra. En lugar de Patrón de Objeto Nulo, el autor ve a este patrón como una generalización del patrón Special Case. Para ilustrar este concepto, se presenta un ejemplo programático en Java utilizando una base de datos y bloqueo de mantenimiento de un sistema de comercio electrónico.
