Contributing to freqtrade:
- Create PRs against the `develop` branch
- New features must have unit tests, conform to PEP8, and be documented
- Discuss feature implementation on Discord or in issues before creating a PR
- Before sending a PR, run unit tests, check for PEP8 compliance, and ensure type-hints are correct
- Core committers prioritize pull requests based on urgency and importance
- Be welcoming to new contributors and follow the Python Community Code of Conduct
